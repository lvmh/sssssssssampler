# PHASE 1 — MOMENT SYSTEM

## Define:

```rust
enum Moment {
    FreezeCut,
    GlitchBloom,
    LockIn,
    PhaseWave,
    Collapse,
    Afterglow,
    UserAccent,
}
```

```rust
struct MomentState {
    active: Option<Moment>,
    timer: u32,
    duration: u32,
    cooldown: u32,
}
```

---

## Rules

* Only ONE moment active at a time
* Moments must:

  * have duration
  * have cooldown (prevent spam)
* Must be deterministic (hash-based triggering)
* Must integrate into existing update loop

# PHASE 2 — MOMENT TRIGGERS

Moments are triggered by:

### Audio:

* transient spikes
* high energy (BUILD / PEAK)

### Parameters:

* rapid knob movement (sample rate, filter especially)

### State transitions:

* entering PEAK
* exiting PEAK (for Afterglow)

---

## Example:

```rust
if transient_spike && energy > 0.8 && moment.cooldown == 0 {
    try_trigger_moment();
```

# PHASE 3 — SIGNATURE MOMENT PACK

Implement EXACTLY these behaviors.

---

## 1. FREEZE-CUT (HIGHEST PRIORITY)

**Feeling:** time snaps

### Behavior:

* Freeze:

  * core motion
  * overlay motion
  * velocity updates
* DO NOT freeze dust
* Apply slight brightness lift (~+10% perceived)

### Duration:

5–20 frames

### Notes:

* Must feel sharp and immediate
* No easing on entry

---

## 2. GLITCH BLOOM

**Feeling:** corruption expands outward

### Behavior:

* Select deterministic seed cell via hash
* Expand glitch radius over time:

  * frame 0: 1 cell
  * frame N: up to 5×5 region
* Use full CHARSET (allow block/box chars)

### Decay:

* Collapse back to original ASCII

---

## 3. LOCK-IN

**Feeling:** alignment / visual clarity

### Behavior:

* One overlay temporarily uses SAME image as core
* Reduce drift amplitude
* Slightly increase opacity

### Duration:

< ½ bar

---

## 4. PHASE WAVE

**Feeling:** smooth ripple through image

### Behavior:

```rust
offset = sin(row * freq + time) * amplitude
```

* Apply horizontal displacement to:

  * core
  * optionally overlays (reduced amplitude)

### Constraints:

* Must preserve readability
* Do not shift beyond bounds

---

## 5. COLLAPSE

**Feeling:** structure breaking apart

### Behavior:

* Temporarily override filter downward
* Remove clusters of core cells (use coherent noise)
* Increase overlay dominance

### Recovery:

* Gradual rebuild using existing filter system

---

## 6. AFTERGLOW

**Feeling:** visual echo / release

### Trigger:

* after Burst / Glitch / Peak exit

### Behavior:

* Increase temporal smearing
* Extend trail persistence
* Slightly reduce motion speed

---

## 7. USER ACCENT

**Feeling:** user caused event

### Trigger:

* rapid parameter delta

```rust
delta = abs(param - prev_param)
```

### Behavior:

* short boost:

  * overlay density
  * motion amplitude
  * glitch probability (within caps)
